# 1) 事実：ワーク・エンゲージメント（WE）の時間変動に関する研究知見

* **短期（⽇次〜週次）には有意な⾃己内変動がある**
  週次ダイアリ研究では、仕事上の出来事や資源の変動に応じて**週ごとにWEが揺れる**ことが繰り返し示されています。例えば、週次の出来事が翌週のWEにも波及する“蓄積・持ち越し”のダイナミクスが報告されています。 ([Onderhouden][1], [PMC][2])

* **中期〜長期（数か⽉〜年）では相対的に安定**
  1年間隔などの**テスト–再テスト信頼性は中〜高程度**で、WEは比較的\*\*“安定した特性（trait）成分”\*\*を持つ、という縦断研究のレビュー・モデルがあります（ただし固定ではなく、変化も起こりうる）。 ([Wilmar Schaufeli][3], [PMC][4])

* **状態（state）としての日内・日次変動も成立**
  日次レベルのレビューでは、個人内の**資源（仕事・個人）変動**と連動して、**日々WEが上下**することがまとめられています。 ([Hogrefe E-Content][5])

> まとめると、**短期は変動性が高く（イベント駆動）**、\*\*中長期は相対的に安定（ベースライン／軌道）\*\*という二面性が確認されています。

# 2) 指標の含意：Momentum と Slope、ウィンドウ長

## 用語の整頓

* **Momentum（差分型）**：例）直近3か⽉平均 − その直前3か⽉平均。\*\*“最近の押し引き”\*\*を強調し、\*\*段差（ステップ）\*\*に敏感。
* **Slope（傾き型）**：回帰傾き（ロバストなら Theil–Sen 等）。\*\*“時間に対するトレンド（勾配）”\*\*を推定。**一方向の緩やかな上昇/下降**に強い。

## 期間選択の考え方（研究事実 → 指標設計）

* **短期の心理状態は週単位で動く**ため、\*\*月次データで“直近のみを強調”\*\*するには

  * **Momentum-3**（直近3−直前3）や、**Slope-3**（3点傾き）は候補。ただし**データ点が少なくノイジー**になりやすい。日/週次ならSlope-3は有効だが、**月次で3点回帰は外れ値に弱い**。
    → \*\*差分（Momentum-3）\*\*は“直近の変化幅”を明快に示せる長所がある。 ([Onderhouden][1], [PMC][2])
* \*\*中期（半期相当）\*\*は、**イベント駆動の揺れを慣らしつつ傾向を出す**観点から、

  * **Slope-6**（6か⽉）が妥当な折衷案。**短期のイベント性**と**中期の安定性**の中間をとり、\*\*ノイズに強いロバスト傾き（Theil–Sen）\*\*が適合。
* \*\*長期（年相当）\*\*は、**構造的な変化／ベースライン**を捉える意図で

  * **Slope-12** が適切。**月次12点**なら回帰の安定性が増し、**季節性に近いゆっくりした漂い**も拾える。長期の相対安定性という文献とも整合。 ([Wilmar Schaufeli][3])

# 3) ご質問への回答

## Q1. 「Slope-6（E\_slope\_6）」は有りか？

**有り**です。

* 研究上、**週次での変動性**と**年次での安定性**が併存するため、\*\*月次運用では6か⽉傾きが“実務的な中庸”\*\*になります。
* **実務判断**：

  * **短期の局所変化**は Momentum-3 が迅速
  * **中期の持続的変化**は **Slope-6（Theil–Sen 推奨）**
  * **長期の構造的変化**は Slope-12
    → “**3・6・12**のマルチスケール”は\*\*理論（時間スケールの二面性）**と**実務（意思決定）\*\*の両立に適合します。 ([PMC][2], [Onderhouden][1], [Wilmar Schaufeli][3])

## Q2. 短期モメンタムの代替として「Slope-3」は？

**可能だが推奨は限定的**。

* **月次3点回帰は不安定**になりやすく、**外れ値・単発イベント**で傾きが乱高下しやすい。
* **差分ベースの Momentum-3**は「**直近の上げ下げ**」を素直に出すため、\*\*短期の意思決定（早期介入）\*\*に向く。
* ただし、\*\*ロバスト推定（Theil–Sen の3点版は実質中央値差）\*\*や、\*\*正則化（スプライン/β制約）\*\*を併用できるなら Slope-3 も検討可。
* 実務上は、**Momentum-3 を主、Slope-3 は補助**が妥当。

以下は、事実（既存研究から知られている一般原理）と、それを踏まえた実務設計（層Cの再定義案）を切り分けて提示します。結論だけ先に言うと——**「強み／弱み」を “組織相対の瞬間風速” でなく、**(1) **長期的プロファイル（Trait）** と (2) **短中期の変化要因（State／Driver）** に分離して扱うのが実用的です。層Cはこの二軸を並立させ、**A（現在位置）× B（軌道）× C（要因）** の三層で説明可能性を高めます。

---

## 1. 事実（一般原理）

* ワーク･エンゲージメント（WE）は\*\*短期には変動（state）\*\*し、\*\*中長期には相対的に安定（trait）\*\*する二面性がある。
* 「組織基準のz（\*\_rating\_z\_org）」は**その月の組織平均・分散に依存**するため、\*\*“組織相対の断面評価”\*\*には有効だが、**個人固有の特性（trait）とは異なる**。
* 「何が変化の要因か」を問うなら、**差分（Δ）や傾き（slope）を各次元（V/D/A）に分解**するのが素直で、**E = V + D + A** なら寄与分解は明確にできる（Eの変化は各次元の和）。

---

## 2. 論点に対する見解

### 論点1：「強み／弱み」を org-z に基づけるのは妥当か？

* **結論**：**“瞬間の組織比較”としては妥当**だが、**個人の本来的な強み／弱み（trait）とは別物**。
* **実務対応**：**層Cを二系統**に分ける。

  1. **C\_trait**（長期プロファイル：個人固有の強み／弱み）
  2. **C\_driver**（短中期の変化要因：今回波での主要ドライバー）

### 論点2：中期的にポジション変化がないなら「強み／弱み」と言えるのでは？

* **結論**：その通り。**6–12ヶ月のロールで一貫して相対的に高低**が続くなら、\*\*その組織コンテクストでの“持ち味”\*\*と解釈して良い。
* **実務対応**：**12ヶ月（最低6ヶ月でも可）のロール平均／分位**でV/D/Aの順位付けを行い、**上位次元＝強み、下位次元＝弱み** として **C\_trait** を出す。

### 論点3：層Bの変動が起きたとき、要因（V/D/A）をどう出すべきか？

* **結論**：**短期（3ヶ月）の差分寄与**と**中期（6ヶ月）の傾き寄与**の**両方でドライバーを特定**するのが実務的。
* **実務対応**：

  * **短期ドライバー**：

    $$
      \Delta E_{3} = \Delta V_{3} + \Delta D_{3} + \Delta A_{3},\quad
      \Delta V_{3} = \text{mean(last3 V)} - \text{mean(prev3 V)}\ (\text{D/A 同様})
    $$

    $|\Delta V_{3}|,|\Delta D_{3}|,|\Delta A_{3}|$ のうち、\*\*大きいもの（複数可）\*\*を **C\_driver\_short** として列挙。
  * **中期ドライバー**：
    **Theil–Sen の `slope_6`** を各次元（V/D/A）にも算出し、符号・大きさで **C\_driver\_mid** を列挙。
  * **報告は “短期/中期の双方”** を出し、**層B（Trend\_B）が「改善中／悪化中」になった根拠**を**透明に説明**する。

## 3. 層Cの再定義（実装設計）

### 3.1 出力カラム構成（提案）

* **C\_trait\_strength / C\_trait\_weakness**

  * 対象：V/D/A
  * **基準**：**12ヶ月ロールの org-分位 or z**（データ量の都合で6ヶ月fallback可）
  * 例：上位分位（≥P70）を**強み**、下位分位（≤P30）を**弱み**。閾値は自動チューニング可。
* **C\_driver\_short**（短期ドライバー）

  * **基準**：$|\Delta V_{3}|,|\Delta D_{3}|,|\Delta A_{3}|$ の寄与比。
  * 例：$|\Delta \text{dim}| \ge \max(|\Delta V_{3}|,|\Delta D_{3}|,|\Delta A_{3}|) \times 0.7$ などの**相対比**、または**絶対閾値**（部門×月のIQRベース）で複数選出。
* **C\_driver\_mid**（中期ドライバー）

  * **基準**：各次元の **`slope_6`（Theil–Sen）**。符号と大きさで選出。
  * 例：部門×月の `slope_6` 分布の上位/下位四分位を**改善/悪化ドライバー**として列挙。
* **C\_stability**（既存どおり）

  * **変動大／要注意**のタグは、**E\_std\_6 / E\_iqr\_6** と **低位連続（E\_low\_streak）** を組み合わせて判定。

> **ポイント**：
>
> * **C\_trait** = 「持ち味」（長期の相対）
> * **C\_driver** = 「今回（短中期）の変化要因」
>   これにより、「今どうか（A）」「どちらへ向かっているか（B）」「なぜそうなっているか（C）」が明確化します。

## 4. しきい値と頑健性

* **C\_trait**：分位（P70/P30）や z（±0.5〜0.7）等を**部門×月の分布で自動調整**（母数が小さい部門は全社分布で補完）。
* **C\_driver\_short**：ベースは **相対寄与比**（トップ寄与の70%基準など）＋**絶対しきい**（IQRベース）で**ノイズを抑制**。
* **C\_driver\_mid**：`slope_6` を**Theil–Sen**で算出（外れ値耐性）。**符号一貫性**（直近2波のサイン一致）を条件に含め、**過剰反応を抑える**。
* **複数要因**：寄与条件を満たす次元は**複数列挙**（ご指摘どおり、複合要因の扱いを標準化）。

  z\_部門 ≤ −1 の割合（部署内基準で低位）。

  直近3ヶ月の押し引き中央値（短期の波）。

* **median\_E\_slope\_6 / median\_E\_slope\_12**

  低位（z\_org≤−1）が2ヶ月以上連続の人の割合。

  最新カテゴリ分布（個人の最新月で集計）。
  行動：

  * 低位・悪化中：最優先介入（役割縮減/支援投入）
  * 低位・改善中：小さな成功の継続支援
  * 中位・悪化中：予防介入（ミスマッチ修正）

# “z” が付く指標（z-score／標準得点）とは

## 定義

**z-score（標準得点）**は、ある観測値 $x$ が、その参照集団の平均から**何標準偏差分**離れているかを表す指標です。
本分析では、参照集団を明示的に限定した **「同月×部門（section）」** と **「同月×部署（group）」** の2種類を使っています。

* **section 基準（例：`Engagement_z_section`）**

  $$
  z^{(\mathrm{section})}_i = \frac{x_i - \mu_{\mathrm{section}}}{\sigma_{\mathrm{section}}}
  $$

  ここで $\mu_{\mathrm{section}}, \sigma_{\mathrm{section}}$ は、**同じ月・同じ部門に属する全メンバー**のその指標（V/D/A/E）の平均・標準偏差。

* **group 基準（例：`Engagement_z_group`）**

  $$
  z^{(\mathrm{group})}_i = \frac{x_i - \mu_{\mathrm{group}}}{\sigma_{\mathrm{group}}}
  $$

  $\mu_{\mathrm{group}}, \sigma_{\mathrm{group}}$ は、**同じ月・同じ部署**内での平均・標準偏差。

> 実装上の注意：標準偏差は **母集団標準偏差（ddof=0）** を用いています。分母ゼロ（標準偏差=0）になる場合は、その月その集団の値が全員同一ということで、**z=0** としています。

## 使い分け（何を基準に相対化するか）

* `*_z_section`：**部門内**での相対位置（同業務文脈で比較 → 公平性・解釈性が高い）
* `*_z_group`：**部署内**での相対位置（少人数ゆえの細かな差も見えるが、ばらつきが小さく不安定になりやすい）

本プロジェクトでは、**部門差が大きい**ことを踏まえ、**section 基準を主軸**にしています。部署が未設定のケースは **group = section** にフォールバックします。

## 解釈の目安（しきい値）

* $|z| \ge 1.0$：その月の同集団で**顕著に高い／低い**水準
* $|z| \ge 0.5$：**中程度の偏位**（「現時点の強み／弱み」の示唆や早期注意喚起に使用）

> 注意：z-score 自体は**正規分布を仮定しません**。ただし「±1 で上位約16%」のような**確率的解釈は正規近似**を前提とする経験則です。分布形が歪な場合は、その解釈は近似に留まります。

## なぜ z-score を使うのか（採用理由）

1. **文脈整合の相対化**：部門や部署ごとの平均・ばらつきに合わせて正規化することで、「A部門での高い」が「B部門でも高い」とは限らない問題を回避。
2. **月次リセット**：月ごとに再計算するため、季節性や制度変更など**時期依存**の影響を軽減。
3. **スケール統一**：V/D/A/E の生点（0–18, 0–54）を、\*\*同一のスケール（標準偏差単位）\*\*で比較できる。

## よくある誤解と限界

* **「zは正規性が必要？」** → 不要です。標準化の計算自体は任意の分布で定義できます。
* **小集団の不安定性**：人数が極端に少ない部署では $\sigma$ が小さく、z が**過大**になりがちです。section 基準を主に使うのは、これを緩和するためです。
* **分布が二峰性／裾が厚い**場合：±1の意味合いが直感とずれることがあるため、\*\*補助的に分位点（例：25/50/75%）\*\*を併記すると頑健です。
* **参照集団が違えば z も変わる**：`*_z_section` と `*_z_group` は別物です。**基準が違えば解釈も別**になります。

## 実務での読み方（本分析の取り決め）

* **位置（Level\_A）**：まず `Engagement_z_section` を主基準（±1）で **高位／中位／低位**。`Engagement_z_group ≤ -1` があれば**低位を優先**。
* **安定性（C\_stability）**：`E_std_6` や `E_iqr_6` が**同月×部署**の上位帯、または `E_low_streak_section ≥ 2` で注意喚起。
* **強み／弱み（スナップショット）**：`{V,D,A}_z_section` の\*\*±0.5\*\*で当月の示唆。
* **特性（12ヶ月）**：`{V,D,A}_z_section_trait_med` を\*\*±0.5\*\*で中長期の得手不得手を把握。
* **トレンド**は z から直接は読まず、**モメンタム／傾き**で判定（z は相対位置の把握に主に使用）。

## 参考：ロバスト代替（必要に応じて）

* 分布が大きく歪む・外れ値が頻発する部署では、

  $$
  z_{\mathrm{robust}} = \frac{x - \mathrm{median}}{1.4826 \times \mathrm{MAD}}
  $$

  のような**ロバスト標準化**（中央値・MAD）への切り替えを検討できます（レポート併記や将来の運用オプションとして）。

## 「ばらつき大」と「散らばり大」の違い（そのままの単語を使用）

* **ばらつき大（std\_E\_z\_section）**

* **両方↓**：**均質**。共通の成功要因を維持・展開。

### 要点だけ：P10 / P90 の意味

- **P10（10パーセンタイル）**
   分布を小さい順に並べたとき、**下位10%の境界**になる値。
   例：100人のスコアがあるとき、**10番目と11番目の間**に相当する水準（補間を含む）。
- **P90（90パーセンタイル）**
   分布の**上位10%の境界**になる値（＝下位90%の上端）。
   例：100人なら**90番目と91番目の間**に相当する水準。

#### なぜ使うか（本プロジェクトでの位置づけ）

- **厳格帯のしきい値**として利用。
  - 低下側の「**低下加速**」は **P10 以下**（強い下振れ側）を含む条件で判定。
  - 上昇側の「**上昇加速**」は **P90 以上**（強い上振れ側）を含む条件で判定。
- **四分位（Q1=25%点, Q3=75%点）よりも外側**を使うため、
   「通常の増減」ではなく、**際立って強い増減**を捉えるのが目的。

#### 実装上の補足（このコードでの計算粒度）

- 指標（`E_slope_6`, `E_momentum_3`, `E_delta_1`）について、**部署×月でメディアン**を取り、その分布に対して **P10 / P90** を計算。
- **標本数が十分（n≥20）**のときは P10 / P90 を厳格帯に採用、**不足時は Q1 / Q3**（25/75%点）を代替。

#### 直観的な例

- ある月の部署ごとの `E_delta_1`（当月−先月）の中央値が多数集まっているとします。
  - **P10 = −1.2**：下位10%の部署は少なくとも −1.2 以上の下落が起きている。
  - **P90 = +1.3**：上位10%の部署は少なくとも +1.3 以上の上昇が起きている。
     → 個人の判定で `E_delta_1 ≤ P10` かつ他の条件も合致すれば「**低下加速**」へ、`≥ P90` なら「**上昇加速**」の候補になります（※最終判定はスロープやモメンタムとのAND/OR条件も含む対称ルール）。

**まとめ**

- **P10/P90 = 分布の10%点/90%点**。
- 「非常に弱い/非常に強い」側のしきい値として、**加速（強い変化）**を普通の変化から切り分けるために使っています。
- データが少ない場合はより安定な **Q1/Q3** を代替にして、過剰反応を防いでいます。

  - `delta1_neg_strict`= **P10**（n≥20 のとき）。不足なら **Q1**。
- 判定
  - **低下加速**：
    - ベースライン＝**低下中** かつ
    - `E_slope_6 ≤ slope6_bad_strict` **AND** (`E_delta_1 ≤ delta1_neg_strict` **OR** `E_momentum_3 ≤ mom3_bad_strict`)
       → 中期の下向きが**強い水準**に到達し、かつ**足元も強い下向き**。
  - **低下継続**：
    - ベースライン＝**低下中** かつ **上記「低下加速」に当てはまらない**（＝下向きだが加速域までは至らない）。

> 対応する上向き側の厳格化（参考）：
>  **上昇加速**＝ `E_slope_6 ≥ slope6_good_strict` AND (`E_delta_1 ≥ delta1_pos_strict` OR `E_momentum_3 ≥ mom3_good_strict`)（good側は P90 / 代替 Q3）。

要点は次のとおりです。

## 用語の位置づけ（どの分位を、どの用途で使うか）

- `*_bad`, `*_good`
   → **Q1/Q3（25/75パーセンタイル）**を基準にした“緩めの”帯。
  - `*_bad` = **Q1**（下位25%境界）
  - `*_good` = **Q3**（上位25%境界）
  - 用途：**ベース判定**（例：上昇/低下ベース、継続系の判定）に用いて、過度な誤判定を避けつつ感度を確保。
- `*_bad_strict`, `*_good_strict`
   → **P10/P90（10/90パーセンタイル）**を基準にした“厳しめの”帯（サンプル数が少ない場合は Q1/Q3 にフォールバック）。
  - `*_bad_strict` = **P10**（下位10%境界；N<20 など少数時は Q1 に置換）
  - `*_good_strict` = **P90**（上位10%境界；N<20 など少数時は Q3 に置換）
  - 用途：**強いシグナルが必要な判定**（例：「上昇加速」「低下加速」など“加速”系）に用いて、偶然的な揺らぎでの誤検出を抑制。

> いずれの分位も、「(wave, group) ごとの**中央値**」をまず作り、その中央値の**全体分布**に対して Q1/Q3/P10/P90 を取っています（＝個人判定で使う“グローバル”な帯）。
>  部署ダッシュボード側の Q1/Q3 は「**同月（wave）×全部署**」の分布に対して都度計算（＝**月内ローカル**）です。

## 直感的な違い

- **Q1/Q3（\*_bad / \*_good）**：
   「下位25%/上位25%」の切り方。**母集団の1/2が“中間帯”**に残るため、**検知しやすい**一方で、**誤検知も相対的に増えやすい**。
- **P10/P90（\*_bad_strict / \*_good_strict）**：
   「下位10%/上位10%」の切り方。**端の10%に入るほど強いシグナル**。**検知は厳格**になり、**偶然のノイズで越えにくい**。

## どの判定でどちらを使っているか（一例）

- **ベースの方向**（Trend_B_base：「上昇中/低下中/安定」）
   → `*_good`/`*_bad`（Q3/Q1）を主に使用。
   理由：トレンドの基調を広めにすくい、月次の連続性（ヒステリシス）でノイズを抑える。
- **直近の向き**（Trend_B_recent：「直近上昇/下降/横ばい」）
   → `E_delta_1` の `*_good`/`*_bad`。
   理由：直近1ヶ月の変化は振れやすいので、Q3/Q1で「上がった/下がった」を相対評価。
- **加速の検知**（Trend_B_refined：「上昇加速／低下加速」）
   → `E_slope_6`, `E_delta_1`, `E_momentum_3`, `E_accel_6` の **`\*_good_strict`/`\*_bad_strict`（P90/P10）をAND**で使用。
   理由：本当に強い上向き/下向きの“加速”だけを拾うため。偶然の一発や薄い傾きでは通さない。

## 数値イメージ（例）

- あるメトリクスのグローバル分布で
  - Q1 = −0.20, Q3 = +0.25
  - P10 = −0.45, P90 = +0.50
- このとき
  - `*_bad`（Q1）＝−0.20 を下回れば「弱い側」
  - `*_good`（Q3）＝+0.25 を上回れば「良い側」
  - `*_bad_strict`（P10）＝−0.45 未満なら「強く悪い」
  - `*_good_strict`（P90）＝+0.50 超なら「強く良い」
- 従って「上昇継続」は Q3 超え程度でも成立し得る一方、「上昇加速」は P90 超え（かつ他の加速項目も P90 超え）でないと通らない。

## サンプル不足時のフォールバック

- `*_bad_strict`/`*_good_strict`（P10/P90）は、**母数が少ないと分位推定が不安定**になるため、実装では **N<20** 等の条件で **Q1/Q3 に置換**しています。
- こうして「厳格帯」の**過度な不安定化を回避**しています。

------

### まとめ

- `*_bad/good`＝**Q1/Q3**（緩い帯、ベース／継続系に使用）
- `*_bad_strict/_good_strict`＝**P10/P90**（厳しい帯、加速など強シグナル系に使用；少数時はQ1/Q3へフォールバック）
- 目的：**検知感度（拾いやすさ）**と**特異度（誤検知抑制）**のバランスを、判定の性質に応じて使い分ける。
