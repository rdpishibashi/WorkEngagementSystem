# ワーク・エンゲージメント（WE）指標と評価・アクション設計（最新版 v3.5.20 準拠）

> 本書は**月次データ**を前提に、個人・部署のワーク・エンゲージメントを定量化し、
>  **現状（層A）×軌道（層B）×安定性・要因（層C）\**を体系化して意思決定へ接続するための設計ドキュメントです。
>  \*\*v3.5.20\*\* の実装方針（Trend_B の \*\*base/recent/refined\*\* と\**対称ラベリング**）に整合させました。
>  しきい値（Q1/Q3・P10/P90）、標準化（z）、組織階層（section/group）の扱いは従来通りです。 

------

## 0. データ前提と共通ルール（不変）

- **入力**：`vigor_rating, dedication_rating, absorption_rating` は整数、`Engagement = V + D + A`（常に一致）
- **時間軸**：`year, month` → `__wave__ = yyyy-mm`
- **組織階層**：`section`（部門）／`group`（部署；空は section で補完）
- **標準化（z）**：同月×部門/部署での平均・母標準偏差（ddof=0）。`*_z_section` を主基準、`*_z_group` を補助に使用。
- **分位点しきい**：通常帯＝**Q1/Q3（25/75%点）**、厳格帯＝**P10/P90**（標本不足時は Q1/Q3 にフォールバック）。
- **ロバスト化**：傾きは外れ値に強い **Theil–Sen**。 

------

## 1. 個人：層A（現在位置）

**指標**：`Engagement_z_section`（主）／`Engagement_z_group`（補）＝同月×集団に対する相対位置（標準偏差単位）。
 **評価**：`z_section ≥ +1.0`＝高位、`≤ -1.0`＝低位、その他＝中位。ただし `z_group ≤ -1.0` の場合は低位を優先。 

**主なアクション**：
 高位＝裁量付与・挑戦機会、中位＝予防フォロー、低位＝面談・負荷/役割調整。

------

## 2. 個人：層B（軌道＝トレンド）

### 2.1 指標と役割

- **短期変化**：`E_delta_1 = E(t) − E(t−1)`（単月差、**直近の向き**）
- **短期モメンタム**：`E_momentum_3 = 直近3ヶ月平均 − 直前3ヶ月平均`
- **中期傾向**：`E_slope_6`（Theil–Sen、最大6点窓）
- **長期補助**：`E_slope_12`（同、最大12点窓）

> 短期イベント性と中期の持続性を分離し、過剰反応と見逃しを同時に防ぐ設計。 

### 2.2 しきい値の推定（部署×月ベース）

- 通常帯：Q1/Q3、厳格帯：P10/P90（標本不足時は Q1/Q3 へフォールバック）。
- **固定定数**：`STRICT_LOW=0.10`／`STRICT_HIGH=0.90`（要件に合わせて変更可）。 

### 2.3 判定の三層構造（**base / recent / refined**）

#### A) **Trend_B_base**（基調）

- `(E_momentum_3 > Q3) ∨ (E_slope_6 > Q3)` ⇒ **上昇中**
- `(E_momentum_3 < Q1) ∨ (E_slope_6 < Q1)` ⇒ **低下中**
- 上記以外 ⇒ **安定**
   ※直近2波ヒステリシスで単月ノイズの影響を抑制。 

#### B) **Trend_B_recent**（直近）

- `E_delta_1 > Q3` ⇒ **直近上昇**／`< Q1` ⇒ **直近下降**／その他 ⇒ **直近横ばい**。 

#### C) **Trend_B_refined**（最終ラベル：最新版 v3.5.20）

- **原則**：
  - **「単月反転」\**は \*\*>0 / <0\*\* の\**符号のみ**で厳密に判断（0 は上昇/下降に含めない）。
  - **継続・加速**は **Q3/Q1** と **P90/P10** の組合せで強度を区別。

| 基調（base） | 直近（recent） | **refined（最新版）**                                        | 意味合い                             |
| ------------ | -------------- | ------------------------------------------------------------ | ------------------------------------ |
| 低下中       | 直近上昇       | **回復**（単月反転：`E_delta_1 > 0` **かつ** `E_delta_1_prev ≤ 0`） | 下向き基調に対する**単月の切り返し** |
| 低下中       | 直近上昇       | **反転上昇傾向**（2連続上げ：`E_delta_1 > 0` **かつ** `E_delta_1_prev > 0`） | 上向き方向への**連続的遷移**         |
| 低下中       | 直近横ばい     | **底打ち**                                                   | 反転可能性の示唆                     |
| 低下中       | 直近下降       | **低下加速**（厳格AND：`E_slope_6, E_delta_1, E_momentum_3, E_accel_6 ≤ P10`）／**低下継続**（通常） | 下げの**強まり**／**継続**           |
| 安定         | 直近上昇       | **上昇期待**                                                 | 上向き芽の形成                       |
| 安定         | 直近横ばい     | **安定維持**                                                 | 横ばい維持                           |
| 安定         | 直近下降       | **低下警戒**                                                 | 下向き芽の警戒                       |
| 上昇中       | 直近上昇       | **上昇加速**（厳格AND：`… ≥ P90`）／**上昇継続**（通常AND：`E_slope_6 > Q3` **かつ** `E_delta_1 > Q3`） | 上げの**強まり**／**継続**           |
| 上昇中       | 直近横ばい     | **安定維持**（ゼロ保持：`E_slope_6 = 0` **かつ** `E_delta_1 = 0` を含む） | 上昇基調の小休止                     |
| 上昇中       | 直近下降       | **反落**（単月反転：`E_delta_1 < 0` **かつ** `E_delta_1_prev ≥ 0`）／**反転低下傾向**（2連続下げ：`E_delta_1 < 0` **かつ** `E_delta_1_prev < 0`） | **下向きへの反転**（単月／連続）     |

> 旧版の「上昇持続」は **安定維持** に再編。厳格条件（P90/P10）や Q3/Q1 の扱いは従来表の方針を踏襲。 

**補助タグ**：**ChangeTag＝「変化大」**（`|E_delta_1| ≥ 6`）。*refined* とは独立に注意喚起として併記。

## 3. 個人：層C（安定性・要因・強み/弱み）

- **安定性（C_stability）**：`E_std_6`・`E_iqr_6` が部署×月の **Q3 以上** ⇒ 変動大／`low-streak` との組み合わせで要注意表示。
- **瞬間の強み/弱み**：`{V,D,A}_z_section` の **±0.5** をしきいに抽出（複数可）。
- **長期特性（Trait）**：`{V,D,A}_z_section` の **12ヶ月ロール中央値**で強み/弱みを抽出。
- **ドライバー（短期/中期）**：`Δ3ヶ月` と `slope_6` で強/弱に分け、介入の的を絞る。

## 4. 部署ダッシュボード（DeptDashboard_MS）詳細

### 4.1 データ粒度・集計単位

- **時間軸**：月次（`__wave__ = yyyy-mm`）
- **組織階層**：`section` = 部門、`group` = 部署（空欄/欠損は **section で補完**し、部署=部門として計上）
- **対象メンバー**：当該 `section × group × wave` に在籍する個人（行）のみ

> 以降の **分位点（Q1/Q3, P10/P90）** は**同一 wave（同月）\**内で\**全部署**の分布に対して求める。厳格帯（P10/P90）は標本不足時に Q1/Q3 にフォールバックする設計（個人側と整合）。

------

### 4.2 部署レベルの基礎指標（算出式と直観）

| 指標名（列）          | 定義（部署 d, 月 t）                                         | 解釈（直観）                                 |
| --------------------- | ------------------------------------------------------------ | -------------------------------------------- |
| `mean_E_z_section`    | 同月×部門標準化 `Engagement_z_section` の**部署平均**        | その部署の**水準**（部門内相対位置の平均）   |
| `share_high_section`  | `Engagement_z_section ≥ +1.0` の**構成比**                   | 高位メンバーの**厚み**                       |
| `share_low_section`   | `Engagement_z_section ≤ -1.0` の**構成比**                   | 低位メンバーの**厚み**（多いほどリスク）     |
| `share_lowstreak_ge2` | 各個人の `low_flag`（z≤-1.0）が **2ヶ月以上連続**の割合      | **慢性低位の滞留**リスク                     |
| `std_E_z_section`     | 部署内 `Engagement_z_section` の**標準偏差**                 | **ばらつき**（均質さ/格差）                  |
| `iqr_E`               | 部署内 `Engagement` 実スコアの **IQR=Q3−Q1**                 | **散らばり**（分布の厚み／外れ値にロバスト） |
| `median_E_delta_1`    | 個人 `E(t)−E(t−1)` の**部署中央値**                          | **直近の方向**（上げ/下げの重心）            |
| `median_E_momentum_3` | 個人モメンタム（直近3ヶ月平均−直前3ヶ月平均）の**部署中央値** | **短期モメンタム**                           |
| `median_E_slope_6`    | 個人 `E_slope_6`（Theil–Sen, ≤6点窓）の**部署中央値**        | **中期持続傾向**                             |
| `median_E_slope_12`   | 個人 `E_slope_12`（Theil–Sen, ≤12点窓）の**部署中央値**      | **長期補助傾向**                             |

**補足**

- `Engagement_z_section` は**同月×部門**で標準化：
   `Engagement_z_section = (E − μ(section,t)) / σ(section,t)`（母標準偏差 ddof=0）。部署比較の“主”基準は部門z。
- `low_flag = (Engagement_z_section ≤ −1.0)`、`low_streak` は個人ごとの連続 True 長。
- `Theil–Sen` は**外れ値に頑健**な傾き推定。個人値の中央値で部署傾向を要約し、**極端値の影響を減衰**。

------

### 4.3 主要ラベルの算出（可読化された運営指標）

#### (1) **Dept_Level**（水準）

- **高水準**：`mean_E_z_section ≥ Q3`（同月×全部署分布で上位四分位）
- **低水準**：`mean_E_z_section ≤ Q1`
- **中水準**：上記以外

> *部門内相対位置の平均*で部署の土台の高さを判断。Q1/Q3 採用は**分布の歪みに頑健**で、閾値の**説明責任**を担保。

#### (2) **Dept_Trend**（方向）

- **上昇**：`median_E_slope_6 > Q3` **かつ** `median_E_delta_1 > Q3`
- **低下**：`median_E_slope_6 < Q1` **かつ** `median_E_delta_1 < Q1`
- **横ばい**：上記以外

> **中期の持続方向（slope_6）\**と\**直近の向き（delta_1）\**をAND結合して\**偽陽性**を抑制。単月ノイズのみ／基調のみでは「上昇/低下」を付与しない。

#### (3) **Dept_Risk**（運営上の警戒）

以下のいずれかが **Q3以上**なら **要警戒**、それ以外は **通常**：

- `share_low_section`（低位構成比が多い）
- `share_lowstreak_ge2`（慢性低位の滞留）
- `std_E_z_section`（ばらつき大）
- `iqr_E`（散らばり大）

> リスクは**厚み（量）×継続（質）×不均質（構造）\**の3軸で捉える複合判定。IQR は\**外れ値に頑健**な散らばり尺度として採用。

------

### 4.4 付加評価列（定義の見える化）

**Dept_Risk の直後**に、次の4列を表示（値に対する**位置評価**）：

- `低位構成比(評価)`：`share_low_section ≥ Q3`→「高」、`≤ Q1`→「低」、その他 空欄
- `連続低位構成比(評価)`：`share_lowstreak_ge2 ≥ Q3`→「高」、`≤ Q1`→「低」、その他 空欄
- `ばらつき(std)(評価)`：`std_E_z_section ≥ Q3`→「高」、`≤ Q1`→「低」、その他 空欄
- `散らばり(IQR)(評価)`：`iqr_E ≥ Q3`→「高」、`≤ Q1`→「低」、その他 空欄

> 「高/低」は**同月×全部署**での相対位置（分位点）に対する**カテゴリ表示**。値自体（実数）はダッシュボード数値列に併記。

------

### 4.5 擬似コード（実装と同義の評価ロジック）

```text
# 1) wave=t の部署粒度で集計
for each dept (section, group) at wave t:
  mean_E_z_section  = mean(Engagement_z_section of members)
  share_high_section = mean(Engagement_z_section >= +1.0)
  share_low_section  = mean(Engagement_z_section <= -1.0)
  share_lowstreak_ge2 = mean(low_streak >= 2)
  std_E_z_section   = std(Engagement_z_section)
  iqr_E             = Q3(Engagement) - Q1(Engagement)
  median_E_delta_1  = median(E_delta_1)
  median_E_slope_6  = median(E_slope_6)

# 2) wave=t 内の全部署で分位点
Q1Q3 = {col: (Q1(col over depts at t), Q3(col ...)) for col in
        [mean_E_z_section, median_E_slope_6, median_E_delta_1,
         share_low_section, share_lowstreak_ge2, std_E_z_section, iqr_E]}

# 3) Dept_Level
if mean_E_z_section >= Q3(mean_E_z_section): Level="高水準"
elif mean_E_z_section <= Q1(mean_E_z_section): Level="低水準"
else: Level="中水準"

# 4) Dept_Trend
if median_E_slope_6 > Q3(median_E_slope_6) and median_E_delta_1 > Q3(median_E_delta_1): Trend="上昇"
elif median_E_slope_6 < Q1(...) and median_E_delta_1 < Q1(...): Trend="低下"
else: Trend="横ばい"

# 5) Dept_Risk（いずれかQ3以上）
risk_flag = any([
  share_low_section >= Q3(share_low_section),
  share_lowstreak_ge2 >= Q3(share_lowstreak_ge2),
  std_E_z_section >= Q3(std_E_z_section),
  iqr_E >= Q3(iqr_E)
])
Risk = "要警戒" if risk_flag else "通常"

# 6) 付加評価列
低位構成比(評価)      = "高" if share_low_section >= Q3(share_low_section)
                      else "低" if share_low_section <= Q1(share_low_section) else ""
連続低位構成比(評価)  = "高"/"低"/""
ばらつき(std)(評価)   = "高"/"低"/""
散らばり(IQR)(評価)   = "高"/"低"/""
```

------

### 4.6 運用の示唆（指標→アクション）

- **「低水準 × 要警戒」**：**低位厚み**／**慢性低位**の増加。**網羅面談**、業務・負荷調整、支援制度の迅速適用。
- **「中水準 × 上昇」**：地合い改善。**成功要因の共有**／施策の**水平展開**。一過性かを来月以降で検証。
- **「高水準 × 低下」**：ハイパフォーマ組織の**疲労/飽和**シグナル。**配分見直し**、目的再定義、ローテーション検討。
- **ばらつき高**：リーダー/役割の**偏在**、フィードバックの**ムラ**。**標準化**・**レビュー頻度**の平準化。
- **散らばり高**：評価レンジの**広がり**（中央値付近の厚み）。**層別施策**（低位:保護、高位:挑戦）を併用。

------

### 4.7 しきい値の見える化と可変パラメータ

- **`Thresholds_MS`** シートに、各指標の **Q1/Q3/P10/P90** と母数（部署×月の件数）を掲示。
- **固定定数**：`STRICT_LOW=0.10`, `STRICT_HIGH=0.90` をスクリプト先頭で一元管理（必要に応じて P15/P85 等へ変更可）。

------

### 4.8 品質保証部（group欠落ケース）の扱い

- `group` 空欄は **`group=section` に補完**して集計（「品質保証部」は **部署=部門** として計算）。
- これにより部署単位のメトリクスと分位点評価が**一貫**して適用可能。

------

## 5. 運用上の注意（抜粋）

- **小集団zの不安定性**：極小部署では `*_z_group` が振れやすい。主に `*_z_section` を用いる。
- **分布の歪み**：±1σは近似。必要に応じ**分位点**を併記し説明責任を担保。
- **外生ショック**：制度/環境変更直後は注記を残す（`DeptComment_MS`）。

### 付録A：指標→アクションの簡易対応表（再掲・用語更新）

| 指標/判定          | 代表条件                                                     | 主アクション                |
| ------------------ | ------------------------------------------------------------ | --------------------------- |
| 低位               | `Engagement_z_section ≤ -1`                                  | 面談、負荷調整、役割再設計  |
| **反転低下傾向**   | `E_delta_1 < 0` & `E_delta_1_prev < 0`（上昇中×直近下降のとき） | 失速要因の是正、短期PDCA    |
| **反落**           | `E_delta_1 < 0` & `E_delta_1_prev ≥ 0`（同上）               | 一時的下振れの原因切分け    |
| **回復**           | `E_delta_1 > 0` & `E_delta_1_prev ≤ 0`（低下中×直近上昇のとき） | 支援継続・効果検証          |
| **反転上昇傾向**   | `E_delta_1 > 0` & `E_delta_1_prev > 0`（同上）               | 上向き遷移の後押し          |
| 低下加速／低下継続 | `base=低下中` ×（厳格AND or 通常）                           | 早期介入／継続支援          |
| 上昇加速／上昇継続 | `base=上昇中` ×（厳格AND or 通常）                           | 挑戦配分・裁量付与          |
| 変動大             | `E_std_6` or `E_iqr_6` ≥ Q3                                  | 平準化・標準化・整流化      |
| 連続低位           | `low-streak ≥ 2`                                             | 網羅面談・健康/環境含め点検 |

------
### 付録B：部門/部署ダッシュボードと個人評価の接続

- 部署 `Dept_Risk` が **要警戒**のとき、当該部署の **個人 C_stability**（ばらつき/散らばり）や **C_short/mid_*（ドライバー）**を優先監視。
- 個人の **Trend_B_refined**（例：反落/回復/反転低下傾向）と部署の **Dept_Trend** の**整合/不整合**を点検し、**メカニズム仮説**（負荷、役割、上位方針変更）へ接続。
