## DeptDashboard\_MS の指標：意味とマネジメントへの使い方

各行は「月×部署」。`DeptLegend_MS` シートにも記載しています（簡易版）。

* **mean\_E\_z\_org**
  同月・全社基準の部署平均（0=全社平均）。
  行動：0未満が継続 → 構造要因（負荷配分、役割設計、支援体制）を点検。0超が継続 → 強みの横展開、人員配置の優先枠。

* **share\_high\_org / share\_low\_org**
  z≥+1（高位）/ z≤−1（低位）の構成比。
  行動：高位比率↑は挑戦機会/権限委譲、低位比率↑は個別面談・業務再設計・OJT/メンタリング。

* **std\_E\_z\_dept**
  部門内の z\_部門 のばらつき。
  行動：ばらつき大→評価/負荷/機会配分の不均衡サイン。職務再設計・標準化やクロスレビュー導入。

* **iqr\_E**
  個票 E の四分位範囲（散らばり）。
  行動：散らばり大→プロセス整流化、オンボード/ナレッジ共有を強化。

* **weak\_dim / weak\_dim\_med**
  V/D/A のうち部署中央値が最も低い次元とその値。
  行動：弱点次元に対する部署横断の学習会/ペアリング/リーダーの振る舞い設計（例：Dedication が弱ければ目的/意義の再接続）。

* **share\_low\_dept**
  z\_部門 ≤ −1 の割合（部署内基準で低位）。
  行動：ローカル課題の濃度。担当マネージャ単位でケース会議、短サイクルの職場改善。

* **median\_E\_momentum\_3**
  直近3ヶ月の押し引き中央値（短期の波）。
  行動：施策の即効性評価。小さな改善を素早く検証する PDCA。

* **median\_E\_slope\_6 / median\_E\_slope\_12**
  6ヶ月/12ヶ月の基調傾向（Theil–Sen）。
  行動：6M>0かつ12M>0 → 施策定着。6M>0で12M≈0 → 過渡期、支援を一時的に厚く。負傾き→構造課題に踏み込む。

* **median\_E\_mean\_6 / median\_E\_std\_6 / median\_E\_iqr\_6**
  直近6ヶ月の水準・変動性。
  行動：水準↑×変動↓が理想。変動↑なら業務波・評価制度・指示の揺れを点検。

* **share\_lowstreak\_ge2**
  低位（z\_org≤−1）が2ヶ月以上連続の人の割合。
  行動：慢性化の兆候。健康面/業務面での早期介入、リソース調整。

* **share\_低位\_悪化中 / share\_低位\_改善中 / share\_中位\_悪化中 / share\_高位\_安定**
  最新カテゴリ分布（個人の最新月で集計）。
  行動：

  * 低位・悪化中：最優先介入（役割縮減/支援投入）
  * 低位・改善中：小さな成功の継続支援
  * 中位・悪化中：予防介入（ミスマッチ修正）
  * 高位・安定：挑戦機会/称賛で定着

---

## どう使い分けるか（管理職支援の例）

1. **モニタリング軸**：`mean_E_z_org` と `median_E_slope_6/12` で「水準×トレンド」を俯瞰。
2. **構造/瞬間の切り分け**：`std_E_z_dept` と `iqr_E`（構造ばらつき） vs. `momentum_3`（短期波）。
3. **重点領域の特定**：`weak_dim` とカテゴリ分布（特に低位・悪化中）。
4. **介入優先順位**：`share_lowstreak_ge2` と `share_low_org` が高い部署から先に。

# “z” が付く指標（z-score／標準得点）とは

## 定義

**z-score（標準得点）**は、ある観測値 $x$ が、その参照集団の平均から**何標準偏差分**離れているかを表す指標です。
本分析では、参照集団を明示的に限定した **「同月×部門（section）」** と **「同月×部署（group）」** の2種類を使っています。

* **section 基準（例：`Engagement_z_section`）**

  $$
  z^{(\mathrm{section})}_i = \frac{x_i - \mu_{\mathrm{section}}}{\sigma_{\mathrm{section}}}
  $$

  ここで $\mu_{\mathrm{section}}, \sigma_{\mathrm{section}}$ は、**同じ月・同じ部門に属する全メンバー**のその指標（V/D/A/E）の平均・標準偏差。

* **group 基準（例：`Engagement_z_group`）**

  $$
  z^{(\mathrm{group})}_i = \frac{x_i - \mu_{\mathrm{group}}}{\sigma_{\mathrm{group}}}
  $$

  $\mu_{\mathrm{group}}, \sigma_{\mathrm{group}}$ は、**同じ月・同じ部署**内での平均・標準偏差。

> 実装上の注意：標準偏差は **母集団標準偏差（ddof=0）** を用いています。分母ゼロ（標準偏差=0）になる場合は、その月その集団の値が全員同一ということで、**z=0** としています。

## 使い分け（何を基準に相対化するか）

* `*_z_section`：**部門内**での相対位置（同業務文脈で比較 → 公平性・解釈性が高い）
* `*_z_group`：**部署内**での相対位置（少人数ゆえの細かな差も見えるが、ばらつきが小さく不安定になりやすい）

本プロジェクトでは、**部門差が大きい**ことを踏まえ、**section 基準を主軸**にしています。部署が未設定のケースは **group = section** にフォールバックします。

## 解釈の目安（しきい値）

* $|z| \ge 1.0$：その月の同集団で**顕著に高い／低い**水準
* $|z| \ge 0.5$：**中程度の偏位**（「現時点の強み／弱み」の示唆や早期注意喚起に使用）

> 注意：z-score 自体は**正規分布を仮定しません**。ただし「±1 で上位約16%」のような**確率的解釈は正規近似**を前提とする経験則です。分布形が歪な場合は、その解釈は近似に留まります。

## なぜ z-score を使うのか（採用理由）

1. **文脈整合の相対化**：部門や部署ごとの平均・ばらつきに合わせて正規化することで、「A部門での高い」が「B部門でも高い」とは限らない問題を回避。
2. **月次リセット**：月ごとに再計算するため、季節性や制度変更など**時期依存**の影響を軽減。
3. **スケール統一**：V/D/A/E の生点（0–18, 0–54）を、\*\*同一のスケール（標準偏差単位）\*\*で比較できる。

## よくある誤解と限界

* **「zは正規性が必要？」** → 不要です。標準化の計算自体は任意の分布で定義できます。
* **小集団の不安定性**：人数が極端に少ない部署では $\sigma$ が小さく、z が**過大**になりがちです。section 基準を主に使うのは、これを緩和するためです。
* **分布が二峰性／裾が厚い**場合：±1の意味合いが直感とずれることがあるため、\*\*補助的に分位点（例：25/50/75%）\*\*を併記すると頑健です。
* **参照集団が違えば z も変わる**：`*_z_section` と `*_z_group` は別物です。**基準が違えば解釈も別**になります。

## 実務での読み方（本分析の取り決め）

* **位置（Level\_A）**：まず `Engagement_z_section` を主基準（±1）で **高位／中位／低位**。`Engagement_z_group ≤ -1` があれば**低位を優先**。
* **安定性（C\_stability）**：`E_std_6` や `E_iqr_6` が**同月×部署**の上位帯、または `E_low_streak_section ≥ 2` で注意喚起。
* **強み／弱み（スナップショット）**：`{V,D,A}_z_section` の\*\*±0.5\*\*で当月の示唆。
* **特性（12ヶ月）**：`{V,D,A}_z_section_trait_med` を\*\*±0.5\*\*で中長期の得手不得手を把握。
* **トレンド**は z から直接は読まず、**モメンタム／傾き**で判定（z は相対位置の把握に主に使用）。

## 「ばらつき大」と「散らばり大」の違い（そのままの単語を使用）

* **ばらつき大（std\_E\_z\_section）**

  * 「同一部署（部門基準の z）」での**全体的な広がり**を**標準偏差**で見ています。
  * **外れ値の影響を受けやすい**ため、**一部の極端なメンバー**がいると大きくなりやすい。
  * 解釈：**個々の点の離れ具合**（平均からの離れ）が大きい状態。

* **散らばり大（iqr\_E）**

  * **生のエンゲージメントE**の\*\*四分位範囲（IQR＝Q3−Q1）\*\*で見ています。
  * **外れ値に頑健**（ロバスト）で、「\*\*真ん中50%\*\*の幅」が広いほど大。
  * 解釈：**大多数（中位層）の幅**が広い状態。

### 使い分けの示唆

* **ばらつき大↑ & 散らばり大↓**：外れ値の影響が強い（**一部の突出**）。個別フォロー・配置見直しの検討。
* **ばらつき大↑ & 散らばり大↑**：部門全体で水準が**二極化/広く分散**。全体施策＋階層別支援を併用。
* **ばらつき大↓ & 散らばり大↑**：中心層の幅は広いが極端は少ない。**共通要因の見直し**（負荷配分や業務設計）を優先。
* **両方↓**：**均質**。共通の成功要因を維持・展開。

### 要点だけ：P10 / P90 の意味

- **P10（10パーセンタイル）**
  分布を小さい順に並べたとき、**下位10%の境界**になる値。
  例：100人のスコアがあるとき、**10番目と11番目の間**に相当する水準（補間を含む）。
- **P90（90パーセンタイル）**
  分布の**上位10%の境界**になる値（＝下位90%の上端）。
  例：100人なら**90番目と91番目の間**に相当する水準。

#### なぜ使うか（本プロジェクトでの位置づけ）

- **厳格帯のしきい値**として利用。
  - 低下側の「**低下加速**」は **P10 以下**（強い下振れ側）を含む条件で判定。
  - 上昇側の「**上昇加速**」は **P90 以上**（強い上振れ側）を含む条件で判定。
- **四分位（Q1=25%点, Q3=75%点）よりも外側**を使うため、
  「通常の増減」ではなく、**際立って強い増減**を捉えるのが目的。

#### 実装上の補足（このコードでの計算粒度）

- 指標（`E_slope_6`, `E_momentum_3`, `E_delta_1`）について、**部署×月でメディアン**を取り、その分布に対して **P10 / P90** を計算。
- **標本数が十分（n≥20）**のときは P10 / P90 を厳格帯に採用、**不足時は Q1 / Q3**（25/75%点）を代替。

#### 直観的な例

- ある月の部署ごとの `E_delta_1`（当月−先月）の中央値が多数集まっているとします。
  - **P10 = −1.2**：下位10%の部署は少なくとも −1.2 以上の下落が起きている。
  - **P90 = +1.3**：上位10%の部署は少なくとも +1.3 以上の上昇が起きている。
    → 個人の判定で `E_delta_1 ≤ P10` かつ他の条件も合致すれば「**低下加速**」へ、`≥ P90` なら「**上昇加速**」の候補になります（※最終判定はスロープやモメンタムとのAND/OR条件も含む対称ルール）。

## **DeptDashboard_MS**評価カラム（部署 × 月）

- **Dept_Level**：部門内相対水準の傾き
  - ルール：`mean_E_z_section` に基づき
    - ≥ +0.30 → **高位寄り**
    - ≤ −0.30 → **低位寄り**
    - それ以外 → **中位**
- **Dept_Trend**：基調と足元からの方向感
  - ルール：`median_E_slope_6` と `median_E_delta_1`
    - 両方 > 0 → **上昇気味**
    - 両方 < 0 → **低下気味**
    - それ以外 → **安定気味**
- **Dept_Risk**：注意ポイントのタグ連結
  - ルール（該当すれば順次付与、"・" で連結）
    - `share_low_section ≥ 0.30` → **低位比率高**
    - `share_lowstreak_ge2 ≥ 0.20` → **連続低位多**
    - `std_E_z_section ≥ 1.00` → **ばらつき大**
    - `iqr_E ≥ 5.00` → **散らばり大**
- **DeptSummary_MS**：`Dept_Level・Dept_Trend・Dept_Risk` の要約結合（空欄は自動除去）

## どの分位を、どの用途で使うか

- `*_bad`, `*_good`
  → **Q1/Q3（25/75パーセンタイル）**を基準にした“緩めの”帯。
  - `*_bad` = **Q1**（下位25%境界）
  - `*_good` = **Q3**（上位25%境界）
  - 用途：**ベース判定**（例：上昇/低下ベース、継続系の判定）に用いて、過度な誤判定を避けつつ感度を確保。
- `*_bad_strict`, `*_good_strict`
  → **P10/P90（10/90パーセンタイル）**を基準にした“厳しめの”帯（サンプル数が少ない場合は Q1/Q3 にフォールバック）。
  - `*_bad_strict` = **P10**（下位10%境界；N<20 など少数時は Q1 に置換）
  - `*_good_strict` = **P90**（上位10%境界；N<20 など少数時は Q3 に置換）
  - 用途：**強いシグナルが必要な判定**（例：「上昇加速」「低下加速」など“加速”系）に用いて、偶然的な揺らぎでの誤検出を抑制。

> いずれの分位も、「(wave, group) ごとの**中央値**」をまず作り、その中央値の**全体分布**に対して Q1/Q3/P10/P90 を取っています（＝個人判定で使う“グローバル”な帯）。
> 部署ダッシュボード側の Q1/Q3 は「**同月（wave）×全部署**」の分布に対して都度計算（＝**月内ローカル**）です。

## 直感的な違い

- **Q1/Q3（\*_bad / \*_good）**：
  「下位25%/上位25%」の切り方。**母集団の1/2が“中間帯”**に残るため、**検知しやすい**一方で、**誤検知も相対的に増えやすい**。
- **P10/P90（\*_bad_strict / \*_good_strict）**：
  「下位10%/上位10%」の切り方。**端の10%に入るほど強いシグナル**。**検知は厳格**になり、**偶然のノイズで越えにくい**。

## どの判定でどちらを使っているか（一例）

- **ベースの方向**（Trend_B_base：「上昇中/低下中/安定」）
  → `*_good`/`*_bad`（Q3/Q1）を主に使用。
  理由：トレンドの基調を広めにすくい、月次の連続性（ヒステリシス）でノイズを抑える。
- **直近の向き**（Trend_B_recent：「直近上昇/下降/横ばい」）
  → `E_delta_1` の `*_good`/`*_bad`。
  理由：直近1ヶ月の変化は振れやすいので、Q3/Q1で「上がった/下がった」を相対評価。
- **加速の検知**（Trend_B_refined：「上昇加速／低下加速」）
  → `E_slope_6`, `E_delta_1`, `E_momentum_3`, `E_accel_6` の **`\*_good_strict`/`\*_bad_strict`（P90/P10）をAND**で使用。
  理由：本当に強い上向き/下向きの“加速”だけを拾うため。偶然の一発や薄い傾きでは通さない。

## 数値イメージ（例）

- あるメトリクスのグローバル分布で
  - Q1 = −0.20, Q3 = +0.25
  - P10 = −0.45, P90 = +0.50
- このとき
  - `*_bad`（Q1）＝−0.20 を下回れば「弱い側」
  - `*_good`（Q3）＝+0.25 を上回れば「良い側」
  - `*_bad_strict`（P10）＝−0.45 未満なら「強く悪い」
  - `*_good_strict`（P90）＝+0.50 超なら「強く良い」
- 従って「上昇継続」は Q3 超え程度でも成立し得る一方、「上昇加速」は P90 超え（かつ他の加速項目も P90 超え）でないと通らない。
