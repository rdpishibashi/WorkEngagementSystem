# ワーク・エンゲージメント評価設計と実装技術文書

## はじめに

本書は、ワーク・エンゲージメント（Work Engagement, 以下 WE）に関する時系列評価と定量分析の実装仕様を網羅的に解説する技術文書である。対象は月次で収集された個人の WE 指標（Vigor, Dedication, Absorption）とし、それらを合成した Engagement 指標を用いて、個人と部署の状態、傾向、安定性の3軸に対し体系的なラベル評価とアクション指針を提供する。処理は Python スクリプト `we_analyzer.py` によって実装され、評価設計は実証研究と統計理論に基づいている。

本稿では、使用指標の定義と意味、分析ロジック、評価基準、アクション設計、理論的裏付けをすべて網羅的に解説し、論文・実務双方に耐え得る構造とする。

---

## 第1章：使用指標と統計的設計思想

### 1.1 WE構成指標
- **Vigor（活力）**
- **Dedication（熱意）**
- **Absorption（没頭）**

上記3因子は Utrecht Work Engagement Scale (UWES; Schaufeli et al., 2002) に基づき、個人の WE を構成する。各スコアは整数値として取得され、Engagement は V + D + A の単純和として定義される。

### 1.2 Engagement 指標と Zスコア
- **Engagement** = V + D + A（整数）
- **Zスコア（部門・部署基準）**：
  - `Engagement_z_section = (E - μ_section) / σ_section`
  - `Engagement_z_group` も同様に部署単位で算出

Zスコアは評価の相対位置づけとラベル化に使用され、部署間・部門間の比較を支える。

### 1.3 時系列特徴量（変化と安定性の指標）

| 指標名            | 算出方法                                       | 目的                             | 採用根拠                                   |
|-------------------|-----------------------------------------------|----------------------------------|--------------------------------------------|
| **Delta-1**       | `E(t) - E(t-1)`                               | 直近の一波変化                    | ノイズ含むが短期変動を捉える               |
| **Momentum-3**    | `Mean(E[-3:]) - Mean(E[-6:-3])`               | 短期変化の押し引き                | 週次変動研究との整合（Onderhouden他）      |
| **Slope-6**       | Theil-Sen 傾き（最大6点）                    | 中期的なトレンド傾向              | ロバスト性と変動幅のバランス（Schaufeli）  |
| **Slope-12**      | Theil-Sen 傾き（最大12点）                   | 長期的トレンド（補助）            | 年単位変動への対応                         |
| **Std-6**         | `標準偏差(E[-6:])`                            | 変動性指標                         | 状態の不安定性を検出                       |
| **IQR-6**         | `Q3(E[-6:]) - Q1(E[-6:])`                     | ロバストなばらつき尺度             | 外れ値に強く、散らばり指標として有効       |

## 第2章：分析処理ロジック（we_analyzer.py）

### 2.1 データ構造と基本処理
- **入力**：個人別の V, D, A スコア、年月、所属情報（section, group）
- **Engagement スコアの生成**：V + D + A
- **__wave__ キー生成**：`year` と `month` から `YYYY-MM` 形式を生成

### 2.2 特徴量の生成
- **個人単位の時系列抽出**：対象者ごとに Engagement 時系列を取得し、上記の特徴量（Slope, Momentum, IQR等）を算出
- **Zスコア標準化**：同月×部門／部署の単位で平均・標準偏差を元に標準化
- **Theil-Sen 傾き**：外れ値の影響を抑えるため、回帰の中位傾きを採用

### 2.3 ロバスト性の確保
- モメンタムやスロープは最大点数で制限（max_len）
- Zスコアの計算は ddof=0（母標準偏差）を使用

## 第3章：評価とラベリング設計

### 3.1 層A：現在位置（水準）
- 判定基準：
  - `Engagement_z_section ≥ +1.0` → **高位**
  - `≤ -1.0` → **低位**
  - その他 → **中位**
- `*_z_group ≤ -1.0` なら強制的に低位へ格下げ（補助基準）

### 3.2 層B：軌道（トレンド）
#### 判定構造：三層分類
- **base**：Momentum-3 と Slope-6 に基づき、上昇中／低下中／安定を判定
- **recent**：Delta-1 による直近の上げ下げ
- **refined**：base × recent の組み合わせで9種以上の詳細ラベルを生成
  - 回復、反落、反転上昇傾向、反転低下傾向、上昇加速、低下継続 等

### 3.3 層C：安定性・構造的特性
- **ばらつき**：`Std-6`, `IQR-6` が Q3 以上 → 「変動大」タグ
- **強み/弱み焦点**：V/D/A の Zスコアで ±0.5 をしきいに抽出
- **Trait判定**：12ヶ月ロール中央値で長期的傾向を評価

## 第4章：部署評価ロジックとダッシュボード

### 4.1 部署単位の指標
- `mean_E_z_section`：部署水準
- `median_E_slope_6`：部署の中期傾向
- `share_low_section`：Zスコア -1.0 以下の構成比
- `iqr_E`：ロバストな散らばり

### 4.2 ラベル化ロジック
- **Dept_Level**：部署平均が Q1/Q3 を超えるかで 高/中/低
- **Dept_Trend**：`slope_6`と`delta_1`のAND → 上昇／低下／横ばい
- **Dept_Risk**：低位構成比・慢性低位・ばらつきが Q3 超で「要警戒」

### 4.3 視覚化と補助表示
- 各指標に対し「高／低」評価を分位点に基づき併記（例：「ばらつき＝高」）
- ダッシュボードにて基準値、件数、変化方向を可視化

|----------------------|--------------------------------------------------------------|------------------------------------------------------|
| **低位**             | 面談・健康確認・役割見直し                                   | z ≤ -1.0、安定的であれば早期介入                     |

## 第7章：補足資料と理論背景の詳細（アップロード資料より）

### 7.1 WE の時間変動に関する理論的知見

- **短期（日次〜週次）の自己内変動**が有意に存在。週次の日誌法研究では仕事上の出来事・資源変動に応じて WE が変動。
- **中期〜長期では相対的に安定**。1年単位の再テスト信頼性は中〜高水準と報告されており、trait 成分を含む。
- **日内・日次の状態的変動も成立**しており、資源やストレッサーとの関連が文献上確認されている。

> ⇒ よって、**短期はイベント駆動で変動性が高く**、**中期〜長期はベースラインとして相対安定**という時間スケールの二面性が明確。

### 7.2 指標選定の根拠：Momentum・Slope・期間の整合

| スケール | 指標       | 採用理由                                                     |
| -------- | ---------- | ------------------------------------------------------------ |
| 短期     | Momentum-3 | 直近の変化幅に敏感（段差に強い）。データ点が少ない場合でも差分が明快。 |
| 中期     | Slope-6    | Theil–Sen によるロバスト回帰で傾向を抽出。短期イベントの揺れをならす。 |
| 長期     | Slope-12   | 構造的な変化、季節性、ベースラインを捉える。12点で安定性が高まる。 |

- Slope-3 は外れ値に敏感で不安定 → Momentum-3 を推奨。Slope-3 は補助的に使用。

### 7.3 実務対応設計の指針

- **3・6・12 のマルチスケール指標**により、短期介入・中期傾向・長期観察が可能
- **基調の変化と瞬間の変化を切り分ける設計**（refined ラベル）で早計な判断を避ける
- **変動とトレンドの非対称性（例：短期改善だが中期横ばい）**には C タグで補完

### 7.4 実装設計への反映提案（設計ガイドより）

- `E_slope_6` を `Momentum-3` と組み合わせて `Trend_B_base` を定義（Slope-12 は補助的）
- しきい値は既定値 + 実データ分布に基づく自動チューニング（Q1/Q3ベース）
- 安定性との連携による誤検出防止：一時的回復に対する慎重な評価

### 7.5 引用・出典

- Onderhouden et al., Weigelt et al.：週次ダイナミクスの研究
- Seppälä et al., Wilmar Schaufeli：中長期安定性
- Hogrefe E-Content：日内変動
- UWES マニュアル：尺度の再テスト信頼性

## 第7章：Trend_B_refined 評価ラベル詳細解説（最新版 v3.5.20）

### 7.1 基本原則

- **単月反転の判定**は `E_delta_1` の符号により行い、ゼロは中立（横ばい）として扱う。
- **継続性や加速度の強度**は、以下のしきい値に基づき区別：
  - 通常帯：Q1 / Q3（25％／75％点）
  - 厳格帯：P10 / P90（10％／90％点）

### 7.2 refined ラベル一覧と判定条件

| 基調（base） | 直近（recent） | refined（最終ラベル） | 判定条件                                               | 解釈・意味合い                       |
| ------------ | -------------- | --------------------- | ------------------------------------------------------ | ------------------------------------ |
| 低下中       | 直近上昇       | 回復                  | `E_delta_1 > 0 and E_delta_1_prev ≤ 0`                 | 単月的な上昇反転、支援継続の効果指標 |
| 低下中       | 直近上昇       | 反転上昇傾向          | `E_delta_1 > 0 and E_delta_1_prev > 0`                 | 明確なトレンド転換                   |
| 低下中       | 直近横ばい     | 底打ち                | recent = 0, base = 低下                                | 変化の停滞、反転の予兆               |
| 低下中       | 直近下降       | 低下加速／継続        | `E_slope_6, E_delta_1, E_momentum_3 ≤ P10`             | 持続的下降、支援必要                 |
| 安定         | 直近上昇       | 上昇期待              | -                                                      | 初動としての上向き兆候               |
| 安定         | 直近横ばい     | 安定維持              | -                                                      | 水準維持                             |
| 安定         | 直近下降       | 低下警戒              | -                                                      | 初動としての下降兆候                 |
| 上昇中       | 直近上昇       | 上昇加速／継続        | `E_slope_6 > Q3 and E_delta_1 > Q3`                    | 成長の継続・加速                     |
| 上昇中       | 直近横ばい     | 安定維持              | `E_slope_6 = 0 and E_delta_1 = 0`                      | 上昇維持状態の一時的停滞             |
| 上昇中       | 直近下降       | 反落／反転低下傾向    | `E_delta_1 < 0 and E_delta_1_prev ≥ 0`, または連続下降 | 成長からの失速                       |

### 7.3 ChangeTag（変化大）

- 判定条件：`|E_delta_1| ≥ 6`
- refined ラベルとは独立に、「急変」の注意タグとして併記。

## 第8章：指標 × アクション 対応マップ（最新版）

| 指標またはラベル | 代表的条件                             | 主なアクション指針                 |
| ---------------- | -------------------------------------- | ---------------------------------- |
| 低位             | `Engagement_z_section ≤ -1`            | 面談、負荷調整、職務再設計         |
| 反転低下傾向     | `E_delta_1 < 0 and E_delta_1_prev < 0` | 失速阻止、即時的フィードバック     |
| 反落             | `E_delta_1 < 0 and E_delta_1_prev ≥ 0` | 一時的下降要因の切り分け           |
| 回復             | `E_delta_1 > 0 and E_delta_1_prev ≤ 0` | 支援策の効果検証、継続判断         |
| 反転上昇傾向     | `E_delta_1 > 0 and E_delta_1_prev > 0` | ポジティブフィードバック、裁量拡大 |
| 低下加速／継続   | P10 条件を満たす                       | 高リスク者への早期介入             |
| 上昇加速／継続   | P90 条件を満たす                       | 挑戦機会の強化                     |
| 変動大           | `E_std_6 ≥ Q3` または `E_iqr_6 ≥ Q3`   | 評価・運営の平準化、安定化施策     |
| 連続低位         | `low-streak ≥ 2`                       | 面談網羅、健康・環境点検           |
