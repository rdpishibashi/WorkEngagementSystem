# ワーク・エンゲージメント分析ロジック（最新版）

更新日時: 2025-11-01 10:15

> 本書は `we_analyzer.py` と同梱の設計ドキュメントです。旧版「カテゴリー評価.md」の内容を、現行実装に合わせて**整合更新**し、再構造化しました。事実（=コードに存在）と補足（運用上の前提/改善余地）を分けて記述します。

## 目的・適用範囲（事実）

本プログラムは、`workengagement.xlsx` を入力として、個人レベルの多面的分析と、部門/グループのダッシュボード集計を生成し、`we_report.xlsx` に以下のシートを出力します。

- Individuals（全履歴の個票）
- LatestIndividuals（最新WAVEの個票）
- DeptDashboard（部門/グループ集計）
- Thresholds（判定ロジックの日本語サマリー）

CLI 既定値：`--input=workengagement.xlsx`、`--output=we_report.xlsx`、`--mid-window=6`。

## 入力仕様・前処理（事実）

必須/推奨列は次のとおりです（存在検査はコード内で実施）。

- 時点：`year` と `month` **または** `date`（`__wave__` に正規化し `%Y-%m` 表現）
- 個人ID：`mail_address` があれば優先、なければ `name` を使用し `__person__` を生成
- 組織：`section`（必須）・`group`（空/NaN/"nan"は `section` を代用し `__group__` 化）
- 属性：`name`, `project`, `grade`（存在すれば出力に含める）
- 評価：`vigor_rating`・`dedication_rating`・`absorption_rating`（合成 `Engagement = V + D + A`）

WAVE単位で `section` と `group` の2軸に対し、`V/D/A/E` の zスコア（`*_z_section`, `*_z_group`）を付与。

## 特徴量（事実）

個人×時系列で昇順に計算します。窓長は `--mid-window`（既定6）を中心に設計。

- Engagement（E）系列の多尺度特徴
  - `E_mean_6`, `E_std_6`, `E_iqr_6`
  - `E_slope_6`, `E_slope_12`（Theil–Sen の頑健傾き推定。点不足時は可能範囲で算出）
  - `E_accel_6`（`E_slope_6` の時点差）
  - `E_momentum_3`（直近3点平均 − 直前3点平均）
  - `E_delta_1`, `E_delta_1_prev`（1期間差分）
- V/D/A の短期変化・傾き
  - `V/D/A_delta_1`
  - `V/D/A_slope_6`（Theil–Sen）


## 判定ロジック（事実）

### A. 水準（Level_A）
- `Engagement` に対する固定閾値：
  - `Thriving` > 43
  - `High` > 32
  - `Moderate`（その他）
  - `Low` < 11
  - `Critical` < 3

### B. トレンド（Trend_B_*）
- **基礎（base）**：`E_slope_6` により、`≥ +0.35`=上昇、`≤ -0.35`=低下、その他=安定。履歴は `MID_MIN_RECORDS+1`（=4点）以上。
- **直近（recent）**：`E_delta_1` の符号・閾値に基づく。
- **精緻（refined）**：`E_momentum_3`、`E_delta_1`/`E_delta_1_prev`、過去6点の極値との関係を組み合わせ、
  `上昇加速/上昇継続/回復期待/回復/復活/上昇期待/低下警戒/低下懸念/悪化/低下危機/低下加速/安定維持` へマッピング。
  強シグナルの代表閾値は `TREND_DELTA_STRONG=5.0`、`TREND_MOMENTUM_STRONG=1.5`。
  
  | **日本語** | **英語**       | **意味ニュアンス**               |
  | ---------- | -------------- | -------------------------------- |
  | 上昇加速   | **surging**    | 急上昇している                   |
  | 上昇継続   | **rising**     | 上昇傾向が続く                   |
  | 悪化       | **worsening**  | 状況が悪化している               |
  | 低下危機   | **severe**     | 危険・危機的水準にある           |
  | 低下加速   | **slumping**   | 急激に落ち込んでいる             |
  | 回復期待   | **hopeful**    | 回復の兆しがある                 |
  | 低下継続   | **declining**  | 低下傾向が続く                   |
  | 回復       | **recovering** | 回復中                           |
  | 復活       | **resurgent**  | 復活・再上昇している             |
  | 低下懸念   | **weakening**  | 力を失いつつある、下がり気味     |
  | 上昇期待   | **improving**  | 改善の兆しがある                 |
  | 低下警戒   | **cautious**   | 注意・警戒が必要な状態           |
  | 安定維持   | **stable**     | 安定を保っている                 |
  | non (高位) | **thriving**   | 活気があり非常に良い             |
  | non (低位) | **critical**   | きわめて低調・要注意（最も危険） |

### C. 急変タグ（ChangeTag）
- `|E_delta_1| ≥ 6.0` を基準に付与。

### D. C系自己相対・相対地位
- 完全安定：`C_stability`（中期窓で E/V/D/A の変動幅 ≈ 0；`C_STABILITY_RANGE_EPS=1e-6`）
- 短期強弱：`C_short_strength/weakness`（個人内 `delta_1` の過去分布p10/p90 とロバストZ；`Z_POS=0.8`,`Z_NEG=-0.8`；最小強度 `SHORT_MIN_DELTA=2.0`）
- 中期強弱：`C_mid_strength/weakness`（`slope_6` の過去分布とロバストZ）
- 特性（セクション基準の地位傾向）：`C_trait_strength/weakness`（`*_z_section` のexpanding分位）
- 時点地位：`C_section_strength/weakness`（その時点の `E_z_section` が ±0.5 超）


## 部門ダッシュボード（事実）

集計単位：WAVE × Section × Group。代表指標：
- `mean_E_z_section`（平均の標準化E）
- `share_high_section`（`E_z_section ≥ +1` の割合）
- `share_low_section`（`E_z_section ≤ -1` の割合）
- `share_lowstreak_ge2`（低位が連続2回以上の割合）
- `std_E_z_section`（分布のばらつき）
- `iqr_E`（四分位範囲）
- 中央値系列：`median_E_momentum_3`, `median_E_delta_1`, `median_E_slope_6` ほか

WAVE内の相対比較として四分位（Q1/Q3）で「高/低」評価列（例：`低位構成比(評価)`・`ばらつき(std)(評価)`・`散らばり(IQR)(評価)`）を生成し、
総合ラベル `Dept_Level`、`Dept_Trend`、`Dept_Risk` に集約します。


## 出力物（事実）

- **Individuals**：ID/属性、V/D/A/E、zスコア、Eの多尺度特徴、Trend_B、ChangeTag、Level_A、C系フラグなど
- **LatestIndividuals**：最新WAVEのみの簡易個票（主にレビュー用の要約列）
- **DeptDashboard**：前節の部門指標＋評価列
- **Thresholds**：主要判定のロジックを日本語で要約した説明表


## 互換性・前提（事実）

- pandas **2.2 以降**の `groupby(...).apply(..., include_groups=False)` を使用（コード該当行: L158, L161, L576）。
  旧版では `TypeError` が発生します。必要に応じて `group_keys=False` 等での代替実装が可能。
- Theil–Sen 傾き推定は外れ値に頑健ですが、WAVE間隔を等間隔とみなすため、欠測が大きい場合は傾きの解釈に注意が必要。

## 改善余地（補足・仮説）

- 閾値のデータ駆動較正（警報過多/過少、再現率/適合率などの観点で最適化）
- 不規則サンプリングへの補正（波形の等間隔仮定を緩める時間スケーリング）
- `engagement_rating` 列が併存する場合のソース・オブ・トゥルースの明確化（E=V+D/A 合算との役割整理）
- 出力ブックにレビュー用レイアウト（条件付き書式や注目列の固定）を追加
